# XqRobotV2 训练指标说明

> 解释训练日志中每个参数的含义、正常范围和异常判断。

---

## 一、总体指标

### Iteration
当前训练轮数。每轮收集数据→更新策略→输出指标。

| 指标 | 含义 | 正常范围 | 异常信号 |
|------|------|---------|---------|
| Total steps | 累计环境交互步数 | 持续增长 | 停滞不变 |
| Steps per second | 采样速度 | 15k-40k（RTX 4090） | < 5k（GPU 被占或代码有性能问题） |
| Iteration time | 每轮耗时 | 0.6-2.0s | > 5s（速度异常） |

---

## 二、策略收敛指标

| 指标 | 含义 | 早期 | 中期 | 收敛 | 异常 |
|------|------|------|------|------|------|
| **Mean entropy loss** | 策略随机性。值越大→探索越多 | 5-6 | 1-2 | 0~-2 | **< -10 → 策略崩了**（输出完全确定） |
| **Mean action std** | 动作标准差。越大→抖动越猛 | 0.5 | 0.3 | 0.2 | < 0.05（策略不动了） |
| **Mean value loss** | 价值函数预测误差 | 0.01 | 0.02-0.05 | →0 | 持续 > 1（价值估计不准） |
| **Mean surrogate loss** | PPO 策略更新幅度 | -0.01~0.01 | 0 附近 | →0 | 剧烈波动（学习率过高） |

**关键判断**：
- entropy 持续下降 + action_std 不变 → **正常收敛**
- entropy **突然从正变负** + action_std → 0 + ep_length → 1 → **策略崩溃**

## 三、奖励指标

所有奖励值已乘 `ctrl_dt`（0.01），即每步期望值。

### 3.1 正向奖励（越大越好）

| 指标 | 满分 | 含义 |
|------|------|------|
| **tracking_lin_vel** | 1.5 | 线速度追踪精度。`exp(-err²/0.3) × 1.5`。err=0 得 1.5，err=0.2m/s 得 ~1.0 |
| **tracking_ang_vel** | 1.5 | 角速度追踪精度。同上 |
| **alive** | 1.0 | 每步存活奖励。恒为 1.0 表示没死 |

### 3.2 负向惩罚（越接近 0 越好）

| 指标 | 含义 | 正常范围 | 异常 |
|------|------|---------|------|
| **lin_vel_z** | 垂直方向速度（跳/掉） | -0.01 | < -0.2（机器人频繁掉高） |
| **ang_vel_xy** | roll/pitch 角速度（摇晃） | -0.02~-0.05 | < -0.3（剧烈摇摆） |
| **base_height** | 偏离目标高度 0.65m | -0.1~-0.2 | < -0.5（蹲太低或站太高） |
| **orientation** | 机身倾斜 `gravity_xy²` | -0.01~-0.05 | < -0.3（严重倾斜） |
| **joint_action_rate** | 腿关节抖动程度 | -0.15~-0.2 | < -0.5（腿在高频抽搐） |
| **wheel_action_rate** | 轮子动作变化幅度 | -0.001~-0.01 | 速度控制下约 -0.005 是正常的 |
| **similar_calf** | 左右腿不对称程度 | -0.01~-0.03 | < -0.1（明显跛行） |
| **tsk** | 髋差动命令跟踪误差 | -0.01~-0.03 | < -0.1 |
| **feet_distance** | 轮距是否在 [0.3,0.6]m | **必须为 0.0** | 非 0 表示轮距超限 |

### 3.3 综合

| 指标 | 含义 |
|------|------|
| **Mean reward** | 所有奖励加权和。50+ 优秀，< 5 有问题 |
| **Mean episode length** | 平均存活步数（/ max_steps）。> 900 优秀，< 100 有问题 |

## 四、训练阶段判断

| iter | entropy | ep_length | 状态 | 行动 |
|------|---------|-----------|------|------|
| 0-200 | 5→2 | 2→20 | 探索期 | 正常，等 |
| 200-1000 | 2→1 | 20→500 | 学习期 | 正常，等 |
| 1000-3000 | 1→0 | 500→900 | 收敛期 | 正常，等 |
| 3000-5000 | 0→-2 | 900+ | 精调期 | 接近上限 |
| **任意轮** | **突然变负** | **恒为 1** | **崩溃** | 停训排查 |

---

## 五、常见异常速查

| 症状 | 可能原因 |
|------|---------|
| ep_length 恒为 1，entropy 崩 | obs dim 不匹配、rewards 过重、腿长 DR |
| lin_vel 追踪一直 < 1.0 | 轮子控制方式、ctrl_dt 太低、orientation 过重 |
| orientation 惩罚持续大 | 默认站姿不对称、髋镜像未约束 |
| feet_distance 非 0 | 轮距超限，可能初始站姿有问题 |
| steps/sec < 5k | GPU 被占、_update_feet_distance 每步调 get_body_pos_w |
| wheel_action_rate > 0.01 | 速度控制下正常，不需处理 |
