# 策略模型格式转换指南

## 概述

强化学习训练产出的模型格式为 PyTorch TorchScript（`.pt`），
RK3588s 上运行时需要 RKNN 格式。本指南说明完整转换流程。

```
训练产出 (.pt)  ──→  ONNX  ──→  RKNN  ──→ 机器人部署
```

---

## 文件说明

| 环节 | 格式 | 文件名示例 | 存放位置 |
|------|------|-----------|----------|
| 训练产出 | `.pt` (TorchScript) | `models/policy_htdw_4438_himloco.pt` | 工作区根目录 |
| 中间格式 | `.onnx` | `models/policy_htdw_4438_himloco.onnx` | 工作区根目录 |
| 部署格式 | `.rknn` | `policy_htdw_4438_himloco.rknn` | `src/sim2real/policy/walk/` |

---

## 转换步骤

### 1. 安装依赖

```bash
# 需要已安装 rknn-toolkit2（针对 RK3588s）
pip3 install rknn-toolkit2

# pt → onnx 步骤需要 PyTorch
pip3 install torch onnx
```

验证安装：

```bash
python3 -c "import torch; print(torch.__version__)"
python3 -c "from rknn.api import RKNN; print('rknn-toolkit2 OK')"
```

### 2. Pt → ONNX

使用 `tools/tools/pt2onnx.py` 脚本：

```bash
cd ~/sim2real_master-master
python3 tools/pt2onnx.py
```

脚本工作流程：
1. 用 `torch.jit.load()` 加载 `.pt` 模型
2. 构造 dummy 输入（batch=1, dim=270）
3. 用 `torch.onnx.export()` 导出为 ONNX
4. 用 `onnx` 库验证 ONNX 模型完整性

**注意事项：**
- 必须使用**固定 batch size**（RKNN 不支持动态 batch）
- 输入维度必须与训练一致（HTDW-4438 为 270 = 45 维观测 × 6 帧堆叠）
- 输出维度为 12（关节动作数）

### 3. ONNX → RKNN

使用 `tools/tools/onnx2rknn.py` 脚本：

```bash
cd ~/sim2real_master-master
python3 tools/onnx2rknn.py
```

脚本工作流程：
1. 创建 `RKNN` 对象
2. 配置目标平台为 `rk3588s`
3. 加载 ONNX 模型（指定固定输入尺寸 `[1, 270]`）
4. 构建 RKNN 模型（不做量化：`do_quantization=False`）
5. 导出 `.rknn` 文件

**注意事项：**
- `load_onnx()` 时必须传 `inputs` 和 `input_size_list` 参数，
  否则动态 batch 会导致加载失败
- 如果要做量化推理加速，可将 `do_quantization` 改为 `True`

### 4. 部署到目标目录

```bash
cp ~/sim2real_master-master/models/policy_from_onnx.rknn \
   ~/sim2real_master-master/src/sim2real/policy/walk/policy_htdw_4438_himloco.rknn
```

### 5. 更新 YAML 配置

修改策略配置文件中的 `policy_name` 字段：

```yaml
# src/sim2real/config/walk/htdw_4438_himloco.yaml
policy_name: "walk/policy_htdw_4438_himloco.rknn"
```

## 模型参数

HTDW-4438（himloco）模型的输入输出参数：

| 参数 | 值 | 说明 |
|------|-----|------|
| `num_single_obs` | 45 | 单帧观测维度 |
| `frame_stack` | 6 | 帧堆叠数 |
| `num_obs` | 270 = 45×6 | 模型输入总维度 |
| `num_actions` | 12 | 动作输出维度 |
| `opset_version` | 11 | ONNX opset 版本 |
| `target_platform` | rk3588s | NPU 平台 |

### 观测结构（45 维）

| 范围 | 维度 | 含义 |
|------|------|------|
| `[0:3]` | 3 | 速度指令 (vx, vy, dyaw) |
| `[3:6]` | 3 | 基座角速度 |
| `[6:9]` | 3 | 投影重力向量 |
| `[9:21]` | 12 | 关节位置误差 |
| `[21:33]` | 12 | 关节速度 |
| `[33:45]` | 12 | 上一步动作 |

## 常见问题

### Q: rknn.load_onnx 报 "input shape not support"

**原因**：ONNX 模型有动态 batch 轴。  
**解决**：在 `tools/tools/pt2onnx.py` 中移除 `dynamic_axes` 参数，
并在 `tools/tools/onnx2rknn.py` 中通过 `input_size_list` 指定固定尺寸。

### Q: 转换后的 .rknn 与之前的 .rknn 大小不同

不同训练版本或不同网络架构的模型产物大小可能不同，
属于正常现象，只要推理引擎能正常加载即可。

### Q: 需要量化加速吗？

如果对推理速度有更高要求，可在 `tools/tools/onnx2rknn.py` 中设置
`do_quantization=True`，但量化后会损失少量精度。

## 转换脚本参考

### tools/pt2onnx.py

位于工作区根目录，核心代码：

```python
model = torch.jit.load(pt_path)
model.eval()
dummy_input = torch.randn(1, 270)
torch.onnx.export(model, dummy_input, onnx_path,
                  export_params=True,
                  opset_version=11,
                  do_constant_folding=True,
                  input_names=["input"],
                  output_names=["output"])
```

### tools/onnx2rknn.py

位于工作区根目录，核心代码：

```python
rknn = RKNN()
rknn.config(target_platform="rk3588s")
rknn.load_onnx(onnx_path, inputs=["input"], input_size_list=[[1, 270]])
rknn.build(do_quantization=False)
rknn.export_rknn(rknn_path)
rknn.release()
```
